# Генератор изображений из GIFT-файла (инструкция)

📌 Коротко: этот скрипт парсит файл в формате GIFT (`questions.gift`) и для каждого вопроса генерирует PNG‑картинку с текстом вопроса и вариантами ответов; если в вопросе есть тег `<img src="@@PLUGINFILE@@/Image/имя.png">`, то эта картинка вставляется между вопросом и вариантами.

---

## 🔧 Требования
- Python 3.8+ (Windows / macOS / Linux)
- Библиотеки: Pillow (PIL) и Click

Установка зависимостей (рекомендуется в виртуальном окружении):

1) Быстро установить напрямую:

Windows / PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install --upgrade pip
pip install Pillow click
```

Linux / macOS:

```bash
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install Pillow click
```

2) Или использовать `requirements.txt` (в корне репозитория):

```powershell
pip install -r requirements.txt
```

(Опционально можно установить numpy, если будете добавлять расширенную обработку изображений.)

---

## 🚀 Первый запуск
1. Поместите `questions.gift` рядом со скриптом `generate_images.py`.
2. Убедитесь, что в папке `Image/` лежат исходные изображения, на которые ссылаются теги `<img ... src="@@PLUGINFILE@@/Image/имя.png">` (если такие есть).
3. Запустите в командной строке (не в Python REPL):

```powershell
python generate_images.py
```

При запуске скрипт запросит у вас имя GIFT‑файла (введите с расширением), значение по умолчанию — `questions.gift` (можете нажать Enter для использования значения по умолчанию).

Примеры:

- Интерактивный запуск (скрипт запросит имя файла):

```powershell
python generate_images.py
# В prompt: Введите имя GIFT файла (с расширением) [questions.gift]: <введите имя или нажмите Enter>
```

- Непосредственная передача параметра (скрипт всё равно покажет prompt, значение по умолчанию будет подставлено):

```powershell
python generate_images.py --in my_questions.gift
# prompt будет предзаполнен my_questions.gift — нажмите Enter, чтобы использовать его
```

По умолчанию:
- Сгенерированные картинки сохраняются в `Image/` (папка создаётся, если отсутствует).
- Создаётся файл `questions_images.gift` рядом со скриптом.

---

## ⚙️ Опции командной строки
- `--in <path>` — путь к GIFT (по умолчанию `questions.gift`).
- `--outdir <path>` — папка для изображений (по умолчанию `Image`).
- `--width <px>` `--height <px>` — размер полотна при рендере (по умолчанию 1200x800).
- `--font <path>` — путь к TTF‑шрифту (например `C:\Windows\Fonts\arial.ttf`).
- `--show-answer` — была опция подсветки правильного ответа, в текущей версии обводки нет (опция остаётся для совместимости).
- `--out-gift <file>` — имя итогового GIFT (по умолчанию `questions_images.gift`).
- `--no-trim` — отключить автоматическую обрезку белых полей (по умолчанию обрезка включена).
- `--trim-pad <px>` — запас (padding) в пикселях при обрезке (по умолчанию 10).

Пример полного запуска:

```powershell
python generate_images.py --in questions.gift --outdir Image --out-gift questions_images.gift --show-answer --trim-pad 12
```

## 🧩 Как скрипт обрабатывает разные сценарии GIFT
- Тег с картинкой: `<img height\="480px" width\="640px" src\="@@PLUGINFILE@@/Image/name.png">` — парсер найдет `src`, найдёт файл в `Image/` или рядом с `.gift` и вставит картинку между вопросом и ответами.
  - Если картинка слишком большая — она **сохраняет свой оригинальный размер**, но будет уменьшена пропорционально, чтобы поместиться в холст (максимальная высота по умолчанию — половина высоты холста).
  - Вставка картинок выполняется по **левому краю** (смещение = `margin`).

- Варианты с префиксом веса: `~%50%Текст` — префикс `%50%` остаётся **в результирующем GIFT**, а на картинке рисуется текст без `%50%`.

- Формат `LHS -> RHS;` (например `=y(t) -> Временная функция;`) — **`LHS ->` и символ `;` остаются в GIFT** (вместо RHS ставится буква A/B/...), а **на картинке ставится RHS** (текст после `->`).

- Вопросы с блоком ответов вида `=A =B =C` (все варианты помечены `=`) — такие варианты **не рисуются** на картинке, а в GIFT остаются **в исходном виде**.

- Итоговый GIFT формат: каждый вопрос заменяется на блок
  ```text
  \r\n</br>
  <img height\="{h}px" width\="{w}px" src\="@@PLUGINFILE@@/Image/qNNN.png">{
    =A
    ~B
    ...
  }
  ```
  где `\r\n` сохраняется буквально, и буквы (A, B, ...) соответствуют вариантам на картинке.

## ✋ Частые проблемы и их решение
- Ошибка `SyntaxError: invalid syntax` при вводе `python generate_images.py --show-answer` — вы запускаете команду **внутри** Python REPL (строки с `>>>`). Закройте REPL (`exit()`), затем выполните команду в оболочке (PowerShell / cmd).
- Pillow не установлен: `ModuleNotFoundError: No module named 'PIL'` — установите `pip install Pillow`.
- Проблемы с кириллицей — укажите TTF шрифт через `--font 'C:\Windows\Fonts\arial.ttf'`.
- Изображение не найдено — убедитесь, что путь в `src` обрабатываемого тега совпадает с фактическим местоположением файла (скрипт ищет рядом с `.gift`, в папке скрипта и в `Image/`).
